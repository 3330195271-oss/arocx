飞书接入教程

把 ARO 连到飞书，不难，关键是应用类型、权限和发布版本别漏掉。

这页是给实际操作的人看的。你只要按顺序完成创建应用、开权限、发布版本、回到 ARO 填写信息这几步，就能把订单同步到飞书多维表格里。

先下载 ARO 打开飞书开放平台

开始前先确认

先把这三件事记住，后面会省很多时间。

ARO 当前飞书同步使用的是飞书开放平台自建应用和多维表格接口。为了让接口能拿到 tenant_access_token 并正常创建、补字段、写记录，应用类型和权限范围要配对。

1. 只能用飞书企业自建应用

现在这套接入走的是企业内部应用接口，不是上架市场应用。创建应用时，直接选飞书开放平台里的企业自建应用最稳。

2. 最省事的方法是自动建表

只填 App ID 和 App Secret，然后在 ARO 里点“自动创建飞书表”，系统会帮你创建多维表格、默认数据表，并自动回填 App Token 和 Table ID。

3. 改完权限后记得发布版本

只是在权限管理里勾选还不够。飞书自建应用通常还要发布版本，组织内应用能力才会真正生效。

操作步骤

按下面 6 步做，一次接好。

如果你不想手动找 App Token 和 Table ID，看到第 4 步就可以先回到 ARO 里点自动建表。

第 1 步：创建飞书企业自建应用

打开飞书开放平台，创建一个企业自建应用。应用名称可以直接用 “ARO 订单同步” 或你自己的团队名称，图标随意，后面都可以改。

第 2 步：复制 App ID 和 App Secret

应用创建好后，在凭证或基础信息页把 App ID 和 App Secret 记下来。ARO 软件里会用到这两个值。

第 3 步：开通多维表格权限

到应用的权限管理里，开通下面这组和多维表格有关的权限。为了避免权限名改版带来的差异，ARO 下面同时写了 scope 名和用途。

第 4 步：发布应用版本

权限勾选完成后，点击发布版本，让新权限在企业内生效。如果这一步没做，ARO 里通常会表现成自动建表失败、拿不到 Token 或写记录失败。

第 5 步：回到 ARO 填应用信息

打开 ARO 首页里的飞书同步页，填入 App ID 和 App Secret。如果想最省事，接下来直接点“自动创建飞书表”。

第 6 步：启用同步并验证

看到 App Token、Table ID 和飞书表格地址自动回填后，确认“启用飞书订单同步”已勾选。然后新增一笔测试订单，看飞书里是否出现记录。

权限清单

这套功能实际会用到哪些权限？

ARO 当前会做这些动作：创建多维表格、读取字段、补齐字段、更新字段名、新增订单记录、更新已有订单记录。为了让这些动作都能跑通，推荐至少开通下面这些权限。

如果你在飞书后台看到的是中文权限组，而不是细粒度 scope，就优先勾选和“多维表格创建、字段管理、记录读写”对应的能力。

推荐开通的权限

  • base:app:create：允许应用创建新的多维表格
  • base:field:read：读取字段列表，检查缺哪些列
  • base:field:create：自动补齐缺失字段
  • base:field:update：把主标题字段统一成“订单标题”
  • base:record:create：新增订单记录
  • base:record:update：发货、归还、转寄时更新原记录

软件里怎么填

飞书同步页每个输入框对应什么？

软件里的字段不是随便起的，和飞书开放平台、多维表格对象是一一对应的。下面这张对照表可以直接照着填。

填写对照

  • App ID：飞书应用的客户端 ID，常见前缀是 cli_
  • App Secret：飞书应用密钥
  • App Token：多维表格应用 ID，自动建表后会自动回填
  • Table ID：要写入的数据表 ID，自动建表后会自动回填
  • 主标题字段：默认保持 订单标题 就行
  • 飞书表格地址：多维表格完整链接，自动建表后也会自动回填

自动建表 vs 手动绑定

两种接法都支持，但我更建议你先用自动建表。

自动建表最省心，也最不容易碰到旧表权限不够、字段缺失、主标题不一致的问题。手动绑定适合你已经有现成表格，且很明确知道自己要接哪一张。

自动建表

只要先填 App ID 和 App Secret，点“自动创建飞书表”，ARO 会创建一张新的多维表格，并自动生成默认数据表和字段。

手动绑定已有表格

如果你坚持接已有表格，需要手动填 App Token 和 Table ID。一般多维表格链接里会出现 /base/<app_token>，表格参数里会带 table=<table_id>。

已有表格的额外注意事项

ARO 会检查字段结构，并在必要时补字段、改主标题字段名。如果你不希望动现有表结构，建议新建一张专门给 ARO 同步使用的表，而不是直接接正在手工维护的旧表。

会同步什么

同步到飞书的字段，和软件里看到的是同一套业务数据。

当前同步的是订单信息和设备相关信息，不会把截图图片上传到服务器，也不会把图片推到飞书。

常见报错

大部分接不上，其实都卡在这几类问题里。

如果你已经照着填了还失败，优先对照下面这张排查表，不用一上来就怀疑软件本身。

获取 tenant_access_token 失败

通常是 App ID、App Secret 填错，或者创建的不是企业自建应用。先回飞书开放平台核对凭证。

自动建表失败

大多是没开 base:app:create，或者开了权限但没发布版本。把权限和发布步骤重新检查一遍。

飞书同步尚未完成配置

说明你虽然勾选了启用，但 App Token 或 Table ID 还是空的。最简单的做法是回软件里点一次“自动创建飞书表”。

已有表格写不进去

说明应用对这张已有多维表格没有访问权限，或者表格不在应用可访问范围内。已有表格手动绑定时，这类问题会比自动建表更常见。

字段名和原表不一样

ARO 会把主标题字段统一成 订单标题，并补齐缺失字段。如果你不希望表结构被调整，建议单独新建一张同步表。

切换账号后飞书配置没了

飞书配置是跟 ARO 账号绑定的，不是跟当前电脑绑定。你换到另一个账号登录后，看不到原来那个账号的飞书配置是正常现象。
